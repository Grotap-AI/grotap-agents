WHY: An architecture audit (see ScanTap-ARCHITECTURE-RECOMMENDATIONS.md §A) found the db-per-tenant model is sound but had gaps. Groundwork already MERGED to master:
- app_registry.subscribe_app now fires `app/subscribed` -> appSchemaProvision worker applies migrations/apps/<slug>/*.sql to the tenant DB.

This case finishes the hardening.

DELIVERABLES:
1) REAL RLS enforcement. Today the backend connects to each tenant Neon DB as the OWNER role, which BYPASSES RLS — so the policies (USING tenant_id = current_setting('app.current_tenant_id')::uuid) do nothing even though the GUC is now set. Fix it: EITHER (a) `ALTER TABLE ... FORCE ROW LEVEL SECURITY` on the tenant tables AND connect as a dedicated NON-owner app role that the policies apply to, OR (b) create a per-tenant non-owner role with the GUC default and switch the stored connection string to it. Implement one coherently across backend/migrations/tenant_template.sql and the app schemas (rfid_pipe, ranch_plan). PROVE isolation: a connection whose GUC is set to tenant A cannot read tenant B's rows; an unset GUC reads nothing.
2) Encrypt tenants.neon_database_url at rest. Add a symmetric key in Doppler (e.g. CONTROL_PLANE_ENC_KEY), encrypt on write in control_plane.create_tenant, decrypt on read in control_plane.get_tenant. Migrate existing rows. The column comment already says "store encrypted in prod".
3) Verify the WorkOS webhook signature in the `organization.created` handler (backend/app/routers/tenants.py ~217) using the WorkOS provider's verification + the signing secret from Doppler; reject unsigned/invalid. Today it is unauthenticated.
4) Add admin endpoint POST /apps/{slug}/reprovision-schema (grotap-admin only) that re-fires `app/subscribed` for a given tenant (backfill), and a small script scripts/backfill_app_schemas.py to re-provision existing subscribed tenants.
5) Collapse the two app catalogs: deprecate/remove the legacy hardcoded APP_CATALOG path in backend/app/routers/apps.py in favor of the DB-backed app_registry (keep one source of truth). Do not break the frontend app list.

ACCEPTANCE: py_compile passes; RLS isolation proven with a test (two tenant ids); webhook rejects a bad signature; existing tenants still load apps + data; conn strings stored encrypted and decrypt correctly.
