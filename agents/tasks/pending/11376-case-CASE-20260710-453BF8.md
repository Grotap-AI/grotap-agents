## Requirements
Create ONE new file: scripts/team2_gate_report.py — a read-only CLI report of
Team 2 (open-model agent) gate-pass performance from the llm_usage_log table, used to
evidence-gate the DEFAULT_TEAM promotion decision (promotion bar: >=70% of runs pass the
verification gate without escalating to Claude).

WHY: the owner decision to promote Team 2 to default requires this evidence; today the
numbers are assembled by hand.

BEHAVIOR:
- argparse with one option: --days N (int, default 30).
- Team 2 telemetry rows are: service = 'fleet-team2'. One agent RUN emits one row per
  escalation attempt; meta is jsonb with meta->>'escalation_step' (1-based attempt number).
  A run is identified by task_key (text). ok (bool) marks whether that attempt passed the
  verification gate. model (text) is the model used for the attempt.
- Section 1 "Attempts by model": SELECT model, count(*) AS attempts,
  count(*) FILTER (WHERE ok) AS passed, ROUND(SUM(cost_usd)::numeric, 4) AS total_usd
  FROM llm_usage_log WHERE service = 'fleet-team2'
  AND ts > now() - ($1 || ' days')::interval GROUP BY 1 ORDER BY attempts DESC.
- Section 2 "Runs": one row per task_key — attempts count, whether ANY attempt passed
  (bool_or(ok)), whether it passed WITHOUT Claude (bool_or(ok) FILTER (WHERE model NOT ILIKE
  '%claude%')), total cost. Use a GROUP BY task_key subquery.
- Section 3 "Promotion summary": total runs, runs passed, runs passed without Claude,
  pass-without-Claude percentage, and the line
  "PROMOTION BAR (>=70% pass without Claude): MET" or "... : NOT MET (n=<runs>)".
  If total runs < 10 also print "(low sample — evidence not yet sufficient)".
- Use asyncio.run(main()); DATABASE_URL from env exactly like scripts/db.py (error to
  stderr + exit 2 when unset).

SCHEMA FACTS (do not guess others): llm_usage_log has ts (timestamptz), service (text),
model (text), task_key (text), case_id (text), cost_usd (numeric/float nullable), ok (bool),
meta (jsonb).

GUARDRAILS (STRICT):
- Create ONLY the one new file named above. Do NOT modify, delete, or reformat ANY existing file.
- Read-only: every SQL statement must be a SELECT. No INSERT/UPDATE/DELETE/DDL anywhere.
- Connect with asyncpg using os.environ["DATABASE_URL"] (injected by Doppler at runtime).
  Never hardcode a connection string or any secret. Follow the connection pattern in
  scripts/db.py (asyncpg.connect, try/finally close).
- No new dependencies: standard library + asyncpg only (asyncpg is already used by scripts/db.py).
- Keep the script under ~150 lines. Output is plain text to stdout (aligned columns or
  pipe-delimited rows are both fine).
- The file must pass: python3 -m py_compile scripts/<filename>.
