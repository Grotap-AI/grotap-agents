# Task: Feature: Platform — Create ONE new file: scripts/pipeline_backlog_report.py — a read-only CLI snapsh

## Context
No additional context provided.

## Requirements
Create ONE new file: scripts/pipeline_backlog_report.py — a read-only CLI
snapshot of the pipeline backlog from the pipeline_cases and pipeline_dispatch_log tables
(control-plane Postgres).

WHY: operators hand-write these queries during every backlog sweep.

BEHAVIOR:
- No arguments needed (argparse optional).
- Section 1 "Cases by status": SELECT status, count(*) FROM pipeline_cases GROUP BY 1
  ORDER BY 2 DESC.
- Section 2 "Open cases by priority": SELECT priority, count(*) FROM pipeline_cases
  WHERE status NOT IN ('done','closed','rejected','archived') GROUP BY 1 ORDER BY 1.
- Section 3 "Oldest 10 open cases": SELECT case_id, status, priority, created_at::date
  FROM pipeline_cases WHERE status NOT IN ('done','closed','rejected','archived')
  ORDER BY created_at ASC LIMIT 10.
- Section 4 "Stale pending dispatches (never started, >20 min old)":
  SELECT case_id, agent_server, created_at FROM pipeline_dispatch_log
  WHERE status = 'pending' AND started_at IS NULL
  AND created_at < now() - interval '20 minutes' ORDER BY created_at ASC.
- Use asyncio.run(main()); DATABASE_URL from env exactly like scripts/db.py (error to
  stderr + exit 2 when unset).

SCHEMA FACTS (do not guess others): pipeline_cases has case_id (text), status (text),
priority (text), created_at (timestamptz), case_data (jsonb) — it has NO title column.
pipeline_dispatch_log has case_id (text), status (text), started_at (timestamptz nullable),
agent_server (text), created_at (timestamptz).

GUARDRAILS (STRICT):
- Create ONLY the one new file named above. Do NOT modify, delete, or reformat ANY existing file.
- Read-only: every SQL statement must be a SELECT. No INSERT/UPDATE/DELETE/DDL anywhere.
- Connect with asyncpg using os.environ["DATABASE_URL"] (injected by Doppler at runtime).
  Never hardcode a connection string or any secret. Follow the connection pattern in
  scripts/db.py (asyncpg.connect, try/finally close).
- No new dependencies: standard library + asyncpg only (asyncpg is already used by scripts/db.py).
- Keep the script under ~150 lines. Output is plain text to stdout (aligned columns or
  pipe-delimited rows are both fine).
- The file must pass: python3 -m py_compile scripts/<filename>.
