[ScanTap Mobile] CS463 reader integration — USB (Android) + BLE + Setup screen + geo-per-tag.

DEPENDS ON case D (the grotap-scantap-mobile Expo app + ReaderService interface must exist). If the repo does not exist yet, do not start — report a hold and wait for case D.

HARDWARE: CSL CS463 reader. See the full spec in C:\1Claude\ScanTap - RFID Inventory Management.md (section 5) — it includes the Android USB Host approach, device_filter (Silicon Labs CP210x vendor-id 4292 / product-id 60000), 115200 8N1, and the CSL low-level byte commands (PREFIX A7 B3, START_INVENTORY 01 02, response header 5A C3, EPC parse). Reuse those constants.

DELIVERABLES (swap into case D's ReaderService interface — keep the simulator as a fallback/dev mode):
1) USB transport (Android only): a custom Expo config plugin + native module wrapping the usb-serial-for-android library (github.com/mik3y/usb-serial-for-android). Requires a custom dev client / EAS build (NOT Expo Go). Implement: enumerate USB serial devices, request permission, open at 115200 8N1, write the CSL start-inventory command, read loop, parse 5A C3 frames → EPC hex. AndroidManifest: uses-feature android.hardware.usb.host + USB_DEVICE_ATTACHED intent-filter + device_filter.xml. Power note in README: tablet cannot power the CS463 (12V external); needs OTG / USB-C↔USB-B.
2) BLE transport (iOS + Android, "any phone"): connect to the CS463 over BLE (react-native-ble-plx via dev client). Discover service/characteristics, send the same start-inventory command, subscribe to notifications, parse EPC frames. Provide graceful fallback when USB is unavailable.
3) Setup screen (Menu 2): choose transport (USB / BLE / Simulator), scan/pick device, connect, and show a live diagnostic — total read, reads in last 10s, current geolocation being assigned, and the location it reconciles to. Persist the chosen reader config.
4) Geo-per-tag: for each tag, record the GPS point where signal was STRONGEST (track max signal_strength per EPC during the scan) and store geo_lat/geo_lng/signal_strength with the tag for later auto-location reconciliation.

ACCEPTANCE: documented EAS dev-client build that includes the native USB module; USB path verified against a CS463 if available, else a clear test harness + simulator parity; BLE path at least builds and connects to a generic device in test; typecheck passes; no secrets committed.
