## Requirements
TEAM2-PILOT-4: force UTF-8 stdout/stderr in scripts/db.py so Windows cp1252 consoles never crash on query output

CONTEXT: scripts/db.py prints query results with print("|".join(...)). On
Windows the default stdout encoding is cp1252, so any result value containing
a character outside cp1252 (observed live 2026-07-15: U+2192 arrow in a
pipeline_cases outcome column) raises UnicodeEncodeError in the cp1252 codec
and the whole query run dies after the data was already fetched. Operators
currently have to remember `export PYTHONIOENCODING=utf-8` by hand.

TASK (single file, nothing else):
1. In scripts/db.py, immediately after the imports, reconfigure the standard
   streams to UTF-8 with replacement fallback:
       for _s in (sys.stdout, sys.stderr):
           if hasattr(_s, "reconfigure"):
               _s.reconfigure(encoding="utf-8", errors="replace")
   (guard with hasattr so redirected/non-console streams and older Pythons
   never break; do not wrap in try/except that hides other errors).
2. Ensure `sys` is imported (it already is — do not duplicate the import).
3. Verify before committing: `python -m py_compile scripts/db.py` exits 0.

GUARDRAILS:
- Touch ONLY scripts/db.py. Do NOT change its SQL handling, argument parsing,
  transaction semantics, --url-env behavior, or output format beyond the
  stream encoding.
- No new dependencies.
