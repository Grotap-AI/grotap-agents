## Requirements
TEAM2-PILOT: fix aider token parsing for k/M-suffixed counts in agents/team-run.sh

CONTEXT: team-run.sh `_parse_aider_output` extracts token counts with
`grep -oE '[0-9,]+ sent'` (and the matching `received` pattern). Aider compacts
large counts to k/M units — e.g. "Tokens: 2.0k sent, 63 received." — so "2.0k"
never matches and INPUT_TOKENS ends up empty; per-attempt telemetry then
under-reports input tokens (observed live 2026-07-10 on agent-20 during the
local-engine smoke: tokens_in=? for a real run).

TASK (single file + its test, nothing else):
1. In agents/team-run.sh, update _parse_aider_output so token counts accept
   plain integers ("12,345"), decimals with a k suffix ("2.0k" -> 2000), and an
   M suffix ("1.2M" -> 1200000). Case-insensitive suffix, round to integer.
   Keep the existing flexible grep/sed philosophy (pattern-based, not
   column-based). Cost parsing must remain unchanged.
2. Extend agents/tests/test_team_run.sh with parser cases covering
   "Tokens: 2.0k sent, 63 received." (expect 2000 / 63),
   "Tokens: 1.2M sent, 1.5k received." (expect 1200000 / 1500), and the
   existing comma format "Tokens: 12,345 sent, 678 received." (expect 12345 / 678).
3. Verify before committing: `bash agents/team-run.sh --verify` prints OK and
   `bash agents/tests/test_team_run.sh` passes.

GUARDRAILS:
- Touch ONLY agents/team-run.sh and agents/tests/test_team_run.sh.
- Do NOT change the result-JSON contract, telemetry payload shape, escalation
  ladder logic, timeouts, or any env/secret handling.
